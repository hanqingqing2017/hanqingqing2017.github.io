    Vue(2)

    如果想绑定动态数据 则需要用v-bind v-bind简写成我们的 :

    Vue Components：Vue组件。HTML页面上，负责接收用户操作等交互行为，执行dispatch方法触发对应action进行回应。

    • dispatch：操作行为触发方法，是唯一能执行action的方法。
      actions：操作行为处理模块。负责处理Vue Components接收到的所有交互行为。包含同步/异步操作，支持多个同名方法，按照注册的顺序依次触发。向后台API请求的操作就在这个模块中进行，包括触发其他action以及提交mutation的操作。该模块提供了Promise的封装，以支持action的链式触发。
    • commit：状态改变提交操作方法。对mutation进行提交，是唯一能执行mutation的方法。
      mutations：状态改变操作方法。是Vuex修改state的唯一推荐方法，其他修改方式在严格模式下将会报错。该方法只能进行同步操作，且方法名只能全局唯一。操作之中会有一些hook暴露出来，以进行state的监控等。
      state：页面状态管理容器对象。集中存储Vue components中data对象的零散数据，全局唯一，以进行统一的状态管理。页面显示所需的数据从该对象中进行读取，利用Vue的细粒度数据响应机制来进行高效的状态更新。
    • getters：state对象读取方法。图中没有单独列出该模块，应该被包含在了render中，Vue Components通过该方法读取全局state对象。
      Vue组件接收交互行为，调用dispatch方法触发action相关处理，若页面状态需要改变，则调用commit方法提交mutation修改state，通过getters获取到state新值，重新渲染Vue Components，界面随之更新。

    flexbox 弹性的盒模型

    flex-direction 方向(父级元素)

    flex-direction：
    主轴与行内轴方向作为默认的书写模式。即横向从左到右排列（左对齐）。
    row-reverse：
    对齐方式与row相反。
    column：
    主轴与块轴方向作为默认的书写模式。即纵向从上往下排列（顶对齐）。
    column-reverse：
    对齐方式与column相反。

    flex-wrap：
    flex容器为单行。该情况下flex子项可能会溢出容器
    wrap：
    flex容器为多行。该情况下flex子项溢出的部分会被放置到新行，子项内部会发生断行
    wrap-reverse：
    反转 wrap 排列。

    flex-flow(flex-direction,flex-wrap)

    flex-flow（row，nowrap）
    弹性盒子里的子集盒子的流动布局方式（水平方向从左到右，盒子平铺整个一行 不折行）
    flex-flow（row，wrap）
    弹性盒子里的子集盒子的流动布局方式（水平方向从左到右，盒子平铺整个一行多出的部分可以折行）
    flex-flow（row，wrap-reverse）
    弹性盒子里的子集盒子的流动布局方式（水平方向从左到右，反转整个盒子的排布{从左下角开始，向右平铺，然后这折行这一行的左上角继续平铺}

    justify-content:center; 水平居中

    align-items:center 垂直居中

    align-content 跟align-items属性值比较相似（多行)

    Share

    HTML5+css3(4)

    复习

    盒子模型

    • 组成：content+padding+border+margin
    • content：width和height—内容的宽度和高度
    • padding：内边距 四个方向
    • border：边框 边框只有宽度没有长度（高度）
    • margin：外边距，盒子与盒子之间的距离，有四个方向 数值可以为负数

    margin的兼容问题

    • 上下排布的两个盒子，上面盒子有margin-bottom值，下面盒子有margin-top值，那么这两个值同时出现时，会发生margin值得叠加
    • 父子元素嵌套关系的时候，如果父级元素没有padding或者border，子级元素设置margin-top值得时候，那么子级元素的这个值会传递给父级元素
      • 给父级元素添加overflow：hidden属性
      • 给父级元素添加border-top值或者padding-top值

    bfc ifc

    bfc就是块级盒子的渲染模式

    块级元素的特点：

    • 从父级元素的左上角排列，每个元素独占一行
    • 可以直接设置盒子模型的所有属性
    • 不设置宽度和高度的时候，宽度是父级元素内容的宽度，高度是其本身内容的高度
    • 可以嵌套所有元素
      • ul和ol里面嵌套的第一层级是li
      • dl里面嵌套的第一层是dt 和dd
      • p标签里面不能嵌套任何块级元素

        IFC就是行内盒子的渲染模式

        行内元素的特点
    • 从父级元素的左上角开始排列，每一个元素都是从左到右在一行显示，达到父级元素最大宽度时候，自动折行
    • 不能设置宽度，高度，设置margin和padding上下值得时候，浏览器可以识别但不起作用
    • 宽度高度默认是本身内容的宽高
    • 行内元素尽量不要嵌套块级元素
    • 行内元素会因为元素之间的空格或者回车影响，在显示的时候，盒子与盒子之见会有间距
    • 行内元素的垂直对齐方式是基线对齐

    行内块级元素的特点：

    • 从父级元素的左上角开始排列，每一个元素都是从左到右在一行显示，达到父级元素最大宽度时候，自动折行
    • 可以直接设置盒子模型属性
    • 在不设置宽高的时候，宽度高度是由内容决定
    • 行内元素会因为元素之间的空格或者回车影响，在显示的时候，盒子与盒子之见会有间距
    • 行内元素的垂直对齐方式是基线对齐

    浮动

    属性值 描述
    left 从左到右依次在一行显示
    right 从右往左依次在一行显示
    none 不浮动
    inherit 从父级元素的该属性中继承

    浮动的最初的作用是为了实现文字围绕盒子的排版

    文档流就是块级元素和行内元素默认的排布方式
    脱离文档流文档流内的正常元素识别不到这个元素了（脱离文档流的元素相当于平行漂浮在文档流之上

    解决文档流元素不识别浮动元素的方式

    • 给浮动元素的父级元素添加固定的高度
    • 给浮动元素的父级元素添加溢出隐藏属性
    • 在最后一个浮动元素后面添加一个块级元素，这个块级元素带有clear:both;属性
      clear清除浮动元素对文档流内元素的影响(可以让文档流内的元素识别到浮动元素的高度)
      • left 清除float为left的影响
      • right 清除float为right的影响
      • both 清除float所有的影响
      • inherit 从父级元素上继承该元素

    带有clear属性的元素需要注意以下几点
    元素需要时块级元素
    元素不能带有浮动属性
    元素必须放在最后一个浮动元素后面
    用户是看不到这个元素的

    • visibility:hidden–盒子看不到，但是盒子所占的大小依然还在
    • display:none–盒子完全从页面消失
    • opacity:0–透明度为0（完全透明）

    • 脱离文档流
    • 设置属性值为left时，浮动元素会一次从父级盒子的左侧向右布局，设置属性值为right时，浮动元素回依次从父级盒子的右侧向左排布

    定位

    相对定位

    元素通过上下左右四个方向的数值改变，来改变该元素在页面上的位置
    参照物是这个元素本身位置
    特点：

    • 参照物是元素本身位置
    • 没有脱离文档流
    • 当top和bottom同时有值时候，top值生效
    • 当left和right同时有值得时候，left生效
    • 任何元素都可以设置相对定位元素
    • 属性值支持负值
    • 相对定位元素位移发生改变，但是元素原来的位置还是会被占用，其他元素还是会正常识别这个元素原来的位置

    box-shadow 盒子投影（阴影）

    第一个值 x轴移动距离 正值向右 负值向左
    第二个值 y轴移动距离正值向下 负值向上
    第三个值 模糊半径 没有负值
    第四个值 投影的投射范围 值越大 阴影范围越大 支持负值（可省略）
    第五个值 阴影的颜色
    第六个值 不写 代表外阴影 inset代表内阴影
    PS：box-shadow是css的属性，所以IE低版本浏览器不支持

    border-radius 边框圆角

    border-radius：10px 20px 30px 40px
    PS：box-shadow是css的属性，所以IE低版本浏览器不支持

    绝对定位

    特点：

    • 脱离文档流
    • 可以设置参照物，参照物必须是父级元素（直系父级）
    • 如果是同一个参照物，所有定位元素都会去找同一个起点
    • 可以设置层级关系

    绝对定位有两种情况

    • 默认参照物是盒子在画布出现的第一个位置
    • 自定义参照物：
      • 参照物须是直系父级元素
      • 参照物需要带有定位属性

    绝对定位元素

    • 有宽度和高度的情况下，top和bottom同时有值，top生效。 left和right同时有值，left生效
    • 没有宽度和高度的情况下，top和bottom同时设置值为0的情况下，会将这个盒子拉大，上下值都起作用，左右同理

    ####脱离文档流的元素都有共同的特点：

    • 不分行内和块级元素，只要脱离文档流就可以直接设置盒子模型的所有属性
    • 宽高不设置的时候，默认是本身内容的宽高

    浮动元素、绝对定位、固定定位元素脱离文档流

    固定定位

    特点

    • 脱离文档流
    • 参照物是浏览器的可视窗口
    • 任何元素都可以设置固定定位

    定元素的层级关系

    • 默认是书写顺序在后的定位元素覆盖在顺序前的定位元素
    • 可以使用z-index属性修改定位元素的层级关系
    • 所有定位元素的z-indx默认值都是一样的
    • z-index值是数字没有单位 ，支持负数
    • z-index一般都是同级元素进行层级比较
    • 当参照物是相对定位或绝对定位的时候，父级元素之间没有z-index值，子集的z-index值会出来比较

    基本选择器

    • 类选择器：将标签属性class的属性值当做选择器使用，在使用时需要在属性值前加一个“.”

      基本语法: 选择器:伪类{}

    盒模型

    组成网页的每一个元素，在页面上的表现形式都是一个矩形（所占的位置是一个矩形），这一个个的矩形都叫做盒子（因为每一个矩形都会装东西）

    盒子模型 ：用来描述每一个元素之间的位置关系，和盒子本身的大小属性

    元素的css属性分为两种：

    • 内置属性：元素天生自带的属性
    • 自定义属性：人为添加给元素的css属性，自定义属性的属性权重大于内置属性

      盒子模型的组成

      盒子模型由外边距，边框，内边距，和内容部分组成，其中外边距，边框，内边距都是4个方向。内容只有宽高
      每一个元素都有自己的内置样式，在编辑页面需要将这些内置样式统一重置，保证页面样式的正常展示
    • 外边距：盒子与盒子之间的距离，用css属性margin表示，margin有四个方向
      • margin-top 上外边距
      • margin-bottom下外边距
      • margin-left 左外边距
      • margin-right 右外边距
        margin值可以是数字、百分比、em、rem、并且margin值支持负数
        margin的写法：
    • margin:10px 20px 30px 40px;上右下左
    • margin:10px 20px 30px;上 左右 下
    • margin:10px 20px; 上下 左右
    • margin:10px; 四个方向的值一致
      外边距的兼容问题
    • 当上面的盒子有下外边距，下面的盒子有上外边距时，两个值只有最大值起作用
    • 子级盒子的上边距会上传给父级盒子（指的是有包含关系的所有盒子），当父级盒子没有内边距或边框的时候，就会出现这个问题
      • 解决方式：
        • 给父级元素添加 overflow:hidden;（溢出隐藏）属性
        • 给父级元素添加内边距或者是边框属性
        • 将子元素的margin-top值变为父级元素的padding-top值【最优选项】

          边框：在内边距和外边距之间，包裹盒子的四个方向的线条

    • 边框有四个方向
    • 边框可以设置颜色
    • 边框可以设置样式
      |属性|描述|
      |–|–|
      |border-top-width|上边框的宽度|
      |border-top-color|上边框的颜色|
      |border-top-style|上边框的样式|
      |border-top:1px solid red|上边框宽度样式颜色|
      |border-width:10px 20px 30px 40px;|上右下左的四个方向的边框宽度|
      |border-width:10px 20px 30px;|上 左右 下的边框宽度|
      |border:10px solid red;|四个方向的宽度 颜色 样式都一致|

    transparent透明

    利用边框来制作三角形

    • 内边距是指边框距离盒子内容的距离 padding有四个方向
      • padding-top 上外边距
      • padding-bottom下外边距
      • padding-left 左外边距
      • padding-right 右外边距
        padding值可以是数字、百分比、em、rem、并且padding值不支持负数
        padding的写法：
    • padding:10px 20px 30px 40px;上右下左
    • padding:10px 20px 30px;上 左右 下
    • padding:10px 20px; 上下 左右
    • padding:10px; 四个方向的值一致
    • 内容（content）分别有两个属性代表
    • 宽度width 数字 百分比 rem em
    • 高度height数字 百分比 rem em
    • 如何计算盒子的大小
      • 盒子的宽度=内容的宽度+左右内边距+左右边框宽度
      • boxwidth=width+（padding-left）+（padding-right）+(border-left-width)+(border-right-width)
      • 盒子的高度=内容的高度+上下内边距+上下边框宽度
      • boxheight=height+（padding-top）+（padding-bottom）+(border-top-width)+(border-bottom-width)

    bfc块级盒子渲染模式

    block formatting context
    块级盒子应该如何在页面中排版？

    • 盒子从父级盒子的左上开始
    • 垂直布局
    • 如果遇到上下margin值的时候，会发生margin的折叠
    • 所有bfc渲染的盒子都是基本流（文档流）内的盒子

    IFC行内盒子渲染模式

    inline formatting context

    • 盒子从父级盒子的左上开始
    • 横向排布，达到父级宽度的最大值，自动折行
    • padding-top，padding-bottom，margin-top，margin-bottom可识别但不起作用

      块级元素的特点

    • 独占一行，垂直排列
    • 可以设置盒子模型的所有属性（width，height，padding，border）
    • 在不设置盒子的宽高时候，宽度是其父级盒子内容的宽度，高度是其本身内容的高度
    • 块级元素可以嵌套其他元素
      • ul、ol第一层级里面只能是li
      • dl第一层级只能是dt和dd
      • p不能嵌套块级元素

        行内元素的特点

    • 在一行显示，从左到右，达到父级元素的最大宽度的时候，自动折行
    • 不能设置宽度和高度，padding和margin的上下值
    • 宽度和高度是由本身内容的宽高决定的
    • 行内元素进行嵌套时，不要嵌套块级元素
    • 行内元素之间如果有空格或者回车，在页面上显示的时候会出现盒子与盒子之间的空隙

    • 可以直接设置盒子模型的所有属性

      • 左对齐 text-align：left；

        • 左对齐 text-align：left；
